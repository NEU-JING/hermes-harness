# 000-exploration — 项目诞生前的探索记录

> 这不是 SDD 流程驱动的产物，而是 hermes-harness 项目概念形成阶段的原始讨论记录。

## 文件说明

| 文件 | 性质 |
|------|------|
| `SDD-通用开发机制-优化方案.md` | 架构总览 + 角色设计 + 落地计划（方案 A） |
| `SDD-通用开发机制-补充方案.md` | 目录约束 + 项目 init/upgrade 机制（方案 B） |

## 价值

这两个文件记录了"为什么这样设计"的思考过程——三层分离的动机、角色 Skill 的粒度选择、AGENTS.md 从流程描述到项目配置的演变。当后续对设计决策有疑问时，可以回到这里查看当时的讨论和权衡。

## 与 `docs/current/` 的关系

`docs/current/prd.md` / `spec.md` / `design.md` 是从这两个草稿中提取、结构化后的**权威当前状态**。本目录是**历史快照**，不会随项目演进而更新。
