# NFR Checklist

> Coder Agent 在实现每个 Task 后自检非功能需求。

---

## 性能

- [ ] 数据库查询无 N+1 问题
- [ ] 大数据量场景有分页或限制
- [ ] 关键路径有性能注释（如预期耗时）
- [ ] 无同步阻塞调用（I/O 密集型场景）

## 安全

- [ ] 用户输入已验证和转义
- [ ] SQL 使用参数化查询（无字符串拼接）
- [ ] 敏感数据不写入日志
- [ ] 认证/授权逻辑无绕过路径
- [ ] 外部 API 调用有超时设置

## 错误处理

- [ ] 所有 API 端点有统一错误响应格式
- [ ] 外部依赖失败有降级方案
- [ ] 错误消息对用户友好（不暴露内部实现）
- [ ] 异常被捕获并记录（不静默吞掉）

## 可维护性

- [ ] 函数/方法 ≤ 50 行
- [ ] 类/模块职责单一
- [ ] 无魔法数字（使用常量）
- [ ] 复杂逻辑有注释
- [ ] 命名清晰（不缩写，除非是通用缩写如 `id`, `url`, `api`）

## 测试

- [ ] 正常流程有测试
- [ ] 异常流程有测试（输入无效、权限不足、依赖失败）
- [ ] 边界条件有测试（空值、最大值、超时）
- [ ] 测试不依赖外部状态（R6）
